# Архитектура проекта

Roomly Staff Management реализован как распределенный монолит.

## Компоненты

1. Frontend.
   Статический сайт на HTML, CSS и JavaScript. Запускается через Vite на порту 5174.

2. Backend.
   Отдельный REST API на Node.js и Express. Запускается на порту 3100.

3. Хранилище данных.
   Структурированный JSON-файл `backend/data/db.json`. Для курсовой работы этого достаточно, так как данные имеют явную модель, есть резервное копирование и валидация.

## Основные сущности

- пользователь;
- сотрудник;
- отель;
- отдел;
- смена;
- задача;
- заявка на отсутствие;
- запись аудита.

## Роли доступа

- `admin` имеет полный доступ;
- `hr` управляет сотрудниками, сменами, заявками и отчетами;
- `manager` управляет сменами и задачами сотрудников своего отеля;
- `staff` видит только свои смены и задачи.

## Совместимость с основным сайтом отеля

Совместимость обеспечивается за счет:

- одинакового REST-подхода;
- общей логики авторизации через токен;
- общих идентификаторов отелей;
- CORS-настроек для frontend основного сайта и frontend панели персонала;
- интеграционного endpoint для проверки связи с основным backend Roomly;
- возможности создавать задачи персоналу на основе бронирований.

## Защита данных

Пароли не хранятся в открытом виде. Для них используется хеширование через PBKDF2. Все защищенные операции требуют токен авторизации. Действия пользователей записываются в журнал аудита.
