# Roomly Staff Management

Отдельный сайт для менеджмента персонала отельного сервиса Roomly. Проект состоит из frontend и отдельного backend REST API. Архитектура сделана как распределенный монолит, чтобы соответствовать требованиям курсовой работы.

## Что реализовано

- REST API на Node.js и Express.
- Авторизация через JWT-совместимый токен.
- Роли пользователей: `admin`, `hr`, `manager`, `staff`.
- CRUD для сотрудников.
- Управление сменами с проверкой пересечений по времени.
- Управление задачами сотрудников.
- Заявки на отсутствие сотрудников.
- Журнал действий пользователей.
- Валидация входных данных.
- Понятные сообщения об ошибках.
- Хранение данных в структурированном JSON-файле.
- Резервное копирование базы данных.
- Проверка совместимости с основным сайтом Roomly.
- Автоматизированные backend-тесты.

## Совместимость с основным сайтом Roomly

Сервис использует те же сущности отелей и тот же подход к REST API, что и основной сайт бронирования Roomly. Для связи с основным backend можно указать переменную окружения:

```bash
MAIN_ROOMLY_API_URL=http://127.0.0.1:3000/api
```

Панель персонала проверяет связь через endpoint:

```text
GET /api/integration/roomly-status
```

Если основной backend Roomly не запущен, панель продолжает работать автономно.

## Финальные роли и разделы

- `staff` видит задачи, бронирования и обращения клиентов, может менять статус своих задач и отвечать клиенту.
- `staff` не видит и не может менять разделы «Смены», «Аудит», управление сотрудниками и настройки отсутствий/выходных.
- `admin` имеет полный доступ ко всем разделам, включая смены, сотрудников, закрытие обращений и журналы действий.

## Новые разделы

- **Бронирования** — заявки с основного Roomly через интеграционный REST API.
- **Обращения** — диалог клиента и сотрудника с возможностью закрытия обращения администратором.
- **Аудит** — Staff-журнал и журнал основного Roomly.
